## Key design decisions
- `uuid` primary keys on all tables
- `users.role` enum: `admin | buyer | partner | agent`
- All monetary values: `numeric(15,2)` in INR
- Plot boundaries: `jsonb` array of `{ lat: number, lng: number }` objects
- RLS enabled on every table; service-role key only inside Edge Functions
- `emi_schedule` rows auto-generated by Edge Function on sale creation
- Agent column security: agents read their commission via a restricted view, not the raw `sales` table

## Architecture principles
- Never bypass RLS at the application layer
- Edge Functions handle all computed writes; client never writes commission_amount directly
- One storage bucket per document type: `plot-documents`, `sale-documents`
- Required indexes: `plots(project_id)`, `plots(status)`, `sales(buyer_id)`, `sales(agent_id)`,
  `emi_schedule(sale_id)`, `emi_schedule(status, due_date)`, `leads(agent_id)`
- Monetary rounding: `ROUND(sale_price * commission_percent / 100, 2)` — decide once, enforce in Edge Function only
